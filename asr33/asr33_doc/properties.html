ASR33 Teletype Properties

Properties with the "asr33_" prefix may be specified on the commandline by removing the prefix. Quotes around values may be required, depending on your shell.

The terminal will search for properties in the following order:

  1. prop=value arguments on the commandline
  2. The first existing file on the commandline
  3. Using the environment variable ASR33_CONFIG
  4. Looking for "asr33rc" in the current directory
  5. Looking for ".asr33rc" in the user's home directory

Only one proerties file will be used - they are not combined.

asr33_ansbak
Specifies a string to use for the Answer-back drum. Typically, this drum generated NUL characters, and that is the default. A maximum of 20 characters is used, with the remaining positions on the drum being left as NUL characters.
asr33_delay
The delay inserted after each character is pasted or otherwise automatically sent to the host, in milliseconds. Default is 100.
asr33_rdr_adv_char
tty_rdr_adv_char
The control character to use for performing a single-character read operation ("advance") on the paper tape reader. This emulates the Teletype feature of a solenoid on the reader that is activated by a handshake signal on the host interface. This needs to match whatever is used on the host (e.g. MDS-800) to trigger the reader advance. Default is 0x0f (Ctrl-O).
asr33_rdr_view
The number of characters before and after the current in the reader tape positioning view. Default is 8. Must be between 1 and 30.
asr33_bell
The audio file to use for the bell sound. This file is buffered in memory, so needs to be small. Default is "bell.wav".
asr33_bell_volume
Specifies the audio volume to use for the bell, in percent. Default is "50".
asr33_lines
Specifies the number of lines to view in the text window. Minimum is 10. Default is 24. Scrolling still allows viewing the entire printed text, this property only affects the view size. Beware of impractical numbers for your screen size.

ASR33 Legacy Option Properties

The following properties accept the values (case-insensitive) "yes", "true", "on", or "1" to enable an option. All other values disable. If the property is not set, the default value is used.

asr33_auto_nl
Enables the atuomatic CR-LF at column 80. Default is on.
asr33_auto_rdr
Enables the paper tape reader to respond to DC1/DC3. Default is on.
asr33_auto_pun
Enables the paper tape punch to respond to DC2/DC4. Default is on.
asr33_cr_on_lf
Enables the atuomatic CR when LF is received. Default is off.
asr33_lf_on_cr
Enables the atuomatic LF when CR is received. Mutually exclusive with 'asr33_cr_on_lf'. Default is off.
asr33_nonprint
Enables the Print/Nonprint option, which adds a button to control whether or not received characters are printed (they are always punched if the punch is on). Default is off.
asr33_parity
Enables generation of parity in the 8th bit of keyboard codes. Default is off.
asr33_even
Selects the state of the 8th bit of keyboard codes. Default is on when parity is enabled (EVEN parity), off when parity disabled (SPACE parity bit).

ASR33 Telnet Properties

asr33_host
The hostname or IP address to connect to. This is the same as the first non-property, non-config-file, argument on the command line.
asr33_port
The port number to connect to. This is the same as the second non-property, non-config-file, argument on the command line.
asr33_server
Sets server mode, where the ASR33 listens on the specified host and port, simulating a standalone case where the ASR33 waits for connections from remote ASR33s. Value is ignored, but must be present ("=" at least) when given on the commandline.
